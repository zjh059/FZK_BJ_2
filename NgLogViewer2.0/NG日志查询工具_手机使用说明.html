NG日志查询工具
手机使用说明

C# WinForms / .NET Framework 4.7.2 / 无第三方包

1. 工具在哪里

D:\A_BJ_HJ_LH\6.30_XC\Tools\NgLogViewer

直接运行：

bin\Release\NG日志查询工具.exe

机台无需联网，不需要下载 NuGet 包。只要已能运行原上位机所需的 .NET Framework 4.7.2，一般即可运行本工具。

2. 难度判断

整体是中等难度。界面和按时间查询不难，难点是避免把 PING 误当成 NG、兼容以后日志文字变化，以及读取正在写入的文件时不锁住主程序。本版本已针对这三点做了保护。

3. 第一次使用

  1. 先启动 NG日志查询工具.exe。
  2. 进入“设置与安全清理”。
  3. 日志目录既可选择包含 info、error、debug 的 logs 根目录，也可直接选择其中一个子目录。
  4. 选择目录后可立即查询；若希望下次自动记住，再点击“保存设置”。设置会保存在 EXE 同目录的 NgLogViewer.settings.ini。
  5. 返回“NG 查询与实时监控”。

默认地址：

D:\A_BJ_HJ_LH\6.30_XC\镀金片焊接\FZK.Shell\bin\Debug\logs

4. 历史NG查询

  1. 开始和结束分别提供“日期日历”和“时间上下选择器”，不用手打完整时间。
  2. 选择分类：工站/接口、底板/导向板比对、焊接/MES、扫码异常、设备/通信、系统异常等。
  3. 可选择治具1或治具2。
  4. 可填写“原因关键字”，例如 工站不匹配、接口返回、MES、重试耗尽。
  5. 可填写任意条码，程序会同时匹配底板码、导向板码、主板码、原因及关联上下文。
  6. 一般勾选“信息/WARN”和“ERROR”；只有必要时才勾选 DEBUG，因为 DEBUG 文件较大。
  7. 点击“开始查询”。若文件较多，可点“中止查询”，已经完成文件的结果会保留；之后点“继续剩余”，会从未完成文件继续。

顶部会显示时间段内的 NG 总数和分类统计。点击任意列标题可切换升序/降序；时间列默认支持完整时间排序。文件名带 _02 或“副本”后缀时，也会严格按文件名中的日期筛选。

5. 实时监控

勾选“实时监控新增NG”后，工具按设定秒数检查新内容。

  • 第一次开启时记录各文件当前长度。
  • 之后只读取新增字节，不重复扫描历史文件。
  • 新增 NG 插入表格第一行。
  • 超过最大显示条数后自动丢弃界面最旧行，磁盘原日志不受影响。

推荐刷新间隔：5～10 秒。它不是越快越好；生产机台以稳定为第一优先。

6. 跨行NG事件聚合与最终结果

新版不会只看带 NG 的一行。它会在很短时间窗口内按主板码聚合相关行，例如：

  1. 工站不匹配：接口返回...
  2. MES校验结束，结果: NG，SP码...
  3. 数据库更新结束，底板码...
  4. 焊接完成，MES=NG...

这些行会合并为一条NG事件，避免重复统计，并补全原因、主板码、底板码和关联上下文。若日志中确实没有写某个条码，对应列仍为空，程序不会猜造。

扫码重试规则：焊接扫码第N次，识别结果: NG 属于中间尝试，不单独计为产品NG；后续最终 MES=OK 时不会误报。若最终 MES=NG、工站不匹配或重试耗尽，真实NG仍会保留。

7. 直接复制和打开原日志

  • 选中一行或按住 Ctrl 多选，点击“复制选中”。
  • 表格内按 Ctrl+C 可复制。
  • 右键可复制当前单元格、NG原因、主板码或完整记录。
  • 双击记录会打开可选择文本的详情窗口，点击“复制全部”。
  • “来源文件”列只显示精简文件名，鼠标停留可看完整路径；点击右侧“打开”可直接打开该日志。
  • 右键还可直接打开原日志，或在资源管理器中定位原日志。
  • 顶部和设置页都有“打开日志文件夹”按钮。

8. 兼容以后日志变化

  • 时间解析允许毫秒位数变化。
  • 支持主板码中包含 +、-、斜杠等现场字符。
  • 即使字段提取失败，也保留整条原始消息和关联上下文。
  • “设置与安全清理”中可修改 NG 识别关键词，例如新增前辈以后使用的新词。
  • NG 必须是独立英文词，因此不会把 PING、PONG 误判成 NG。
  • 若未来日志完全改成 JSON、CSV 或改变时间位置，则应修改 LogParser.cs。

8. 清空和清理的区别

操作影响建议
清空当前显示只清工具表格内存，不删日志运行中可安全使用
关闭工具再打开释放全部界面内存数据很多时可用
删除过期历史日志真正删除早于指定天数的 .log 文件停机或低负载时使用

磁盘清理有二次确认，至少保留7天，并且永远保护今天的日志。即便如此，生产过程中仍不建议执行磁盘清理。

9. 为什么不会明显影响机台

  • 用共享只读方式打开日志，不独占、不锁住主程序。
  • 整个工具进程低于普通优先级。
  • 历史查询和实时读取都在低优先级后台线程执行，完成后恢复线程池状态。
  • 实时刷新有防重入保护：上一轮未结束时不会再启动一轮。
  • 每处理一批内容会短暂让出 CPU。
  • 按日志文件日期预筛选，不读取无关日期。
  • 默认不读取体积最大的 DEBUG。
  • 结果显示有上限，避免 DataGridView 无限增长。
  • 实时模式采用增量读取。
  • 同一日志存在“副本”时按事件跨文件去重。

任何程序都无法承诺“绝对零资源占用”。如果现场工控机配置很低，请把刷新设为10秒、最大显示设为1000，并不要勾选 DEBUG。

10. CSV导出

查询完成后点击“导出CSV”，可用 WPS 或 Excel 打开。新版使用 UTF-8 BOM，并写入 sep=, 分隔提示；完整时间和所有条码强制按文本输出，避免时间显示成 57:46.1、条码变科学计数法或内容被截断。导出还包含关联上下文和原始行号。
