# NG日志查询工具

当前版本：1.3.1

- 技术：C# WinForms，.NET Framework 4.7.2
- 依赖：仅系统自带程序集，无 NuGet 包、无网络依赖
- 默认日志：`D:\A_BJ_HJ_LH\6.30_XC\镀金片焊接\FZK.Shell\bin\Debug\logs`
- 运行文件：`bin\Release\NG日志查询工具.exe`
- 使用说明：`NG日志查询工具_手机使用说明.html`

安全设计：查询以共享只读方式打开文件；实时模式只读新增字节；后台线程低优先级；默认不删除磁盘日志；清理功能保护今天的日志并要求二次确认。

1.1新增：跨行NG事件聚合、工站/接口分类、直接复制、右键定位原日志、日期与时间分开选择、列头升降序、WPS/Excel兼容CSV、FI应用图标、低优先级后台实时读取、防重入和跨文件去重。

1.2新增：严格按文件名中的日期筛选（兼容 `_02` 和“副本”后缀）、支持直接选择 info/error/debug 子目录、可中止并继续剩余文件、来源文件精简显示和行内“打开”按钮、查询条件无需先保存即可立即生效，以及更清晰的界面配色。

1.3修正：排除“焊接扫码第N次识别为NG、后续重试成功且最终MES=OK”的中间过程误报；最终MES=NG、工站不匹配等真实终态仍会保留。表格NG内容统一使用红色粗体，蓝底白字表头不变。窗体改为标准 `Form.cs + Form.Designer.cs + Form.resx` 结构，可在 VS2022 设计器中拖动控件微调。工程外壳恢复 VS 标准 `Microsoft.Common.props` 导入和 `Any CPU` 映射，已实际验证 F5 启动及断点命中。

1.3.1修正：`NG`左右紧挨字母或数字时视为产品码内容，不再把 `DR8HV9A02NG...` 这种实际扫码/MES均为OK的记录误报；MES响应明确返回“产品码不存在/產品碼不存在”时，统一归入“焊接/MES”，并显示精简原因。
