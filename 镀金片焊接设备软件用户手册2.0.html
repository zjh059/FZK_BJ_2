1. 一页快速上手

先记住：软件里的“停止设备”只停止上位机的 PLC 轮询并关闭相关服务，不等于急停。出现人员危险、夹伤风险、机械臂异常运动、冒烟或异味时，应立即使用设备的物理急停/总停，并按整机安全规程处理。

停机与交接
  1. 检查现场。急停已按规定复位，安全门/防护罩正常，机械臂运动区域无人，治具和扫码窗口无异物。
  2. 使用指定账号登录。不要自行试建账号；当前版本输入不存在的用户名时可能自动注册新用户。
  3. 等待硬件初始化完成。初始化失败先看提示并点“重试”；普通操作员不得点“跳过”进入生产。
  4. 进入“开始”页。确认欧姆龙 PLC、①/②上下扫码器、Epson 扫码器、EPSON 机械臂均显示“已连接”。必要时点“刷新状态”。
  5. 核对生产前关键值。D0-D5 应处于待机状态，D110 心跳应符合现场约定，①/②焊接次数不得超过已批准上限。
  6. 点击“启动设备”。按钮变为“停止设备”，运行日志出现启动/监控信息后，才允许按工艺流程上料。
  7. 生产中盯三处。扫码记录的 OK/NG、硬件连接文字、设备运行日志。出现 NG、断线、超时或记录缺失，立即停止放行并隔离在制品。
  8. 下班前正常停机。等待当前循环结束并确认机械机构处于安全位置，再点“停止设备”；完成交接记录后，硬件断电按整机电气/机器人规程执行。
质量红线：当前程序在 SFC 关闭时会跳过 MES 查询并把焊接校验按通过处理。生产配置必须由工艺/质量/软件负责人确认 SFC 已启用；操作员不得为“临时生产”关闭 SFC。

2. 安全、范围与权限

本手册说明上位机软件的使用方法，不替代整机电气手册、机械臂安全手册、锁定挂牌（LOTO）制度或现场工艺文件。

2.1 软件实际负责什么

  • 监控 PLC 触发信号和心跳，执行治具 1、治具 2 的扫码、焊接结果处理和清零流程。
  • 连接五个扫码器、欧姆龙 PLC 和 EPSON 机械臂，并显示连接状态。
  • 显示治具焊接次数、扫码记录、运行日志和机械臂状态区域。
  • 在启用 SFC 时查询 MES 测试结果；管理底板/导向板/主板码绑定和用户数据。

2.2 软件没有的安全功能

没有独立软件急停

界面中没有急停、安全门复位或安全互锁按钮。必须使用设备实体安全回路。

没有暂停/复位按钮

当前运行页只有启动/停止、刷新状态、清空日志。不要在手册中寻找不存在的“暂停”或“复位”。

停止不会取消已在执行的一轮

点击停止主要停止后续 PLC 轮询；若某次扫码/MES请求已进入执行，不能把软件停止当作即时运动制动。

无独立报警中心

当前版本主要通过连接状态、扫码记录和运行日志暴露异常，现场应把 PLC/整机报警作为第一依据。

2.3 建议权限矩阵

角色允许操作禁止或需审批
操作员登录、开始页监控、启动/停止、刷新状态、查看记录、切换用户修改设置、改 PLC 地址、删绑定、清计数、关闭 SFC、跳过初始化
班组长/质量确认 NG 隔离、批准重试/返工、审核交接记录未经工艺批准修改通信或结果规则
设备维护员安全停机后检查通信、镜头、配置、日志与硬件绕过安全回路、直接伪造 OK 值

3. 登录与初始化

3.1 登录

  1. 启动程序后，在登录页输入由负责人分配的用户名和密码。
  2. 点击“登录”。密码错误时会提示登录失败；请核对大小写和输入法，不要连续尝试他人账号。
  3. 登录成功后进入硬件初始化阶段。若需要换班，使用顶部“切换用户”，不要多人共用同一账号。
账户风险：当前程序在用户名不存在时会自动创建用户；用户名为 admin 时还可能被赋予管理员角色。生产电脑应由管理员预先创建并保管账号，操作员发现“注册成功”提示时应停止操作并上报，而不是继续生产。

3.2 硬件初始化

初始化成功

系统自动进入主界面。进入“开始”页后仍需核对每个硬件的“已连接”文字，不能只看是否进入页面。

初始化失败

记录屏幕提示；确认设备已上电、网络和接线正常后点“重试”。连续失败则停止生产并联系维护。

“跳过”不是模拟模式：初始化页的“跳过”只会打开主界面，不会建立虚拟 PLC、虚拟扫码器或虚拟机械臂。普通操作员不得用“跳过”绕过硬件故障投入生产。

5. 班前检查与启动

5.1 班前检查表

  • 查看上一班交接记录，确认没有未关闭故障、未隔离在制品或待处理 NG。
  • 确认设备周围无人员、工具和散料，机械臂运动区域无人进入。
  • 确认急停、安全门、防护罩等实体安全装置按整机点检要求正常。
  • 确认治具 1、治具 2 的实体标识与软件 ①/②一致。
  • 清洁扫码器镜头和条码经过区域；不得使用会刮伤镜头的材料。
  • 确认电脑、PLC、扫码器、机械臂和网络设备已按规定上电。
  • 确认当前产品、工单、站点和 SFC/MES 配置属于本批次。
  • 确认 D108/D109 次数未达到更换或保养阈值。

5.2 软件启动步骤

  1. 使用分配账号登录，等待初始化成功。
  2. 进入“开始”页，先不要放入首件。
  3. 逐项确认 PLC、①上、①下、②上、②下、Epson 扫码器和 EPSON 机械臂均为“已连接”。
  4. 点击“刷新状态”，确认无错误弹窗，D0-D5 处于合理待机值，D110 心跳符合 PLC 约定。
  5. 核对焊接次数和 MES/SFC 生产配置，必要时由班组长或质量人员复核。
  6. 点击“启动设备”，确认日志出现设备启动、PLC 读取定时器开始等信息。
  7. 用批准的首件执行一轮，核对扫码记录、条码、OK/NG 与 PLC/设备动作一致。首件未确认不得批量生产。
启动成功判定：硬件全部已连接；无初始化/通信错误；启动按钮已变为“停止设备”；日志持续更新；首件记录与实物条码一致；工艺规定的 MES/SFC 校验有效。

6. 正常生产操作

6.1 操作员标准循环

  1. 放入工件。按工艺确认方向、到位和治具编号，不遮挡条码。
  2. 等待 PLC 触发。治具 1 使用 D0/D1/D2，治具 2 使用 D3/D4/D5。上位机只在信号由 0 变 1 的上升沿启动对应流程。
  3. 观察扫码记录。核对时间、治具、底板码、导向板码、主板码、结果和备注是否属于当前工件。
  4. 确认结果。OK 才能按工艺进入下一步；NG、空码、记录缺失或条码不一致必须隔离。
  5. 监控次数与连接。每批次切换、发生卡料/重扫、连接颜色改变时重新确认。

6.2 扫码比对流程（D0 / D3）

  • 下扫码器读取多个码并识别底板码与主板码；上扫码器读取导向板码。
  • 系统按配置的次数和延迟自动重试；识别成功后，在数据库中核对底板码与导向板码绑定关系。
  • 流程完成后写入“扫码完成”和“扫码结果”地址，并在扫码记录中增加一行。
  • 若顶部扫码器未配置，代码会跳过底板/导向板比对。生产现场必须确认顶部扫码器确实需要并已配置。

6.3 焊接结果流程（D1 / D4）

  • 下扫码器读取底板码与主板码，识别成功后写入“焊接扫码完成”。
  • SFC 启用时，软件向 MES 查询主板码测试结果；MES/网络超时或返回失败应按 NG 处理。
  • 系统更新主板测试结果、底板使用次数，并将焊接结果写给 PLC。
  • 禁止关闭 SFC 来绕过 MES。当前代码在 SFC 关闭时会把 MES 校验直接视为通过。

6.4 清零流程（D2 / D5）

  • 清零由 PLC 触发，不是当前界面上的手动按钮。
  • 系统先用对应下扫码器识别底板码，再清除数据库计数并把 D108/D109 写为 0。
  • 清零会影响治具寿命追踪，只有已批准的更换/保养流程才允许触发；记录清零前值、底板码、执行人和原因。

6.5 机械臂扫码

以当前代码为准：收到机械臂 RobAsc 到位命令后，系统等待约 200 ms，触发机械臂扫码器并按码长/有效性回复 OK 或 NG。当前流程没有调用 MES 报站、没有新增扫码记录，也没有完整更新右侧机械臂状态字段。因此“最新报站结果”不可作为 MES 过站证明。

6.6 生产中禁止事项

  • 不得在设备运动或流程执行中拔插 PLC、扫码器、机械臂或网络线。
  • 不得修改 PLC 地址、扫码规则、MES URL、站点参数或用户角色。
  • 不得删除绑定关系、手工伪造 OK、清除计数后继续使用未保养治具。
  • 不得用“跳过初始化”或关闭 SFC 来恢复生产。
  • 不得仅凭界面颜色放行；必须核对结果文字、条码和实际工件。

7. 异常、报警与恢复

统一处置原则：停止继续上料 → 确保人机安全 → 隔离当前及可疑工件 → 记录时间/治具/条码/画面 → 通知班组长或维护 → 经确认后恢复。

7.1 人身或设备危险

  1. 立即使用实体急停或整机规定的安全停止手段。
  2. 不得进入机械臂区域、跨越安全门或用软件按钮代替能量隔离。
  3. 由具备资格的人员按整机/机器人 SOP 排除危险并复位。
  4. 恢复前检查在制品位置、PLC 信号、扫码记录和机械臂状态，防止重复动作。

7.2 软件 NG 或记录异常

扫码失败/空码

隔离工件；安全停机后检查条码清晰度、位置和镜头；记录原始码与重试次数。不得手输或复制上一件条码。

底板与导向板比对失败

隔离当前工件并核对实体配对；由质量/工艺确认绑定关系。禁止现场直接改数据库使其通过。

MES NG/超时/断网

停止放行并隔离从最后一次确认成功之后的在制品；保留主板码、时间和日志。不得关闭 SFC。

记录缺失或与实物不符

按未验证处理；不要重复触发造成重复报站/重复计数。由维护根据数据库和日志确认。

7.3 连接异常

  • PLC 未连接：停止生产；确认 PLC 上电和网络，由维护检查 IP、端口和通信。不要反复点击启动。
  • 扫码器未连接：记录具体是①上、①下、②上、②下还是 Epson；安全停机后由维护检查供电、网络、触发指令和镜头。
  • 机械臂未连接：不要进入运动区或自行重启控制器；按机器人厂家规程处理。
  • D110 心跳异常：停止继续上料并同时检查 PLC/上位机双方；仅凭“刷新状态”恢复不代表故障已解除。

7.4 软件卡死、崩溃、停电或电脑重启

  1. 先按整机规则保证机构与人员安全，标记在制品位置和状态。
  2. 记录故障时间、最后一条成功记录、当前条码和 PLC/机器人状态。
  3. 重启软件后必须重新登录、重新初始化并核对所有连接。
  4. 查询数据管理和磁盘日志，确认故障前动作是否已写入；不确定时按 NG/待判处理。
  5. 由质量/工艺决定返工、重扫或报废，不得直接从中断步骤盲目续跑。

8. 停机与交接班

8.1 正常软件停机

  1. 停止上料，等待当前工件循环完成；确认没有正在扫码/MES查询/机械臂响应。
  2. 核对最后一件的条码、结果和备注，处理全部 NG 与待判品。
  3. 确认机械机构处于整机规定的安全停机位置。
  4. 点击“停止设备”，确认日志记录停止信息，按钮变回“启动设备”。
  5. 需要关闭程序时再关闭窗口；硬件断电顺序严格按整机电气和机器人手册执行，禁止带电拔线。

8.2 交接班清单

  • 当前账号已退出或切换，未把管理员账号留给下一班。
  • 产品/工单/批次、治具 1/2 当前次数已记录。
  • 本班 NG、重试、MES 超时和断线次数已说明。
  • 所有隔离品均有条码、时间、原因和责任人标识。
  • 未关闭故障、临时措施和待维护项目已书面交接。
  • 关键日志/截图已保存，程序目录剩余空间正常。

9. 数据、日志与追溯

9.1 扫码记录怎么读

字段含义检查要点
时间记录创建时间应与当前工件操作时间一致
治具治具1、治具2或其他流程来源必须与实物位置一致
类型底部+顶部扫码、焊接结果扫码、清零等确认没有把不同流程混淆
底板/导向板/主板本次识别出的码值空值、重复、错位均要隔离
结果0=未测试、1=OK、2=NGPLC完成信号与业务结果不是同一概念
备注比对成功、重试耗尽、MES OK/NG、系统异常等排查时优先查看

9.2 三种信息不要混淆

  • 页面运行日志：用于当班快速观察，可被“清空日志”或定时逻辑清空。
  • 磁盘日志：位于程序安装/运行目录下的 logs/info、logs/error、logs/debug、logs/trace；按日期和大小归档。
  • 数据库/数据管理：保存绑定和业务数据；界面“最大记录数”只限制运行页显示条数，不等于数据保存周期。
报修时不要只发一句“软件坏了”。至少提供发生时间、治具、工件码、界面截图、错误弹窗全文和同一时间段的 info/error 日志。

10. 设置与维护员操作

普通操作员到此停止。本章所有修改均应在安全停机、参数已备份、变更有审批并安排首件验证的前提下进行。

10.1 配置与日志实际位置

  • 系统配置保存在程序当前运行目录下的 Config 文件夹，不是原手册所写的“我的文档\FZK\Config”。
  • NLog 日志保存在程序目录下的 logs 文件夹，不是“我的文档\FZK\Logs”。
  • 备份时应完整复制 Config 文件夹并记录软件版本、日期和设备编号；禁止直接删除配置文件。

10.2 设置页分类

分类主要内容修改风险
IP/软件设置MES测试URL、工位上报URL、工位编码、设备IP、SFC、制程/工站校验、运行间隔关闭SFC会绕过MES；URL或站点错误会造成质量追溯错误

保存配置时，程序会把治具 1 上扫码器的部分通用参数（延迟、触发指令、重连次数和延迟）同步到其他扫码器。修改后应关闭并重新启动软件，使构造时读取的配置全部生效，并完成连接测试、扫码测试和首件确认。

10.3 数据管理

  • 底板绑定：维护底板与导向板绑定，可新增、编辑、删除、刷新和筛选。
  • 主板绑定：维护底板/导向板/主板数据，可能包含批量删除及 Excel/CSV 导出。
  • 用户管理：维护用户名、密码和角色。

删除和批量删除属于不可逆高风险操作。执行前导出备份，记录筛选条件、记录数、执行账号和审批单号；完成后用已知样本复核查询与生产扫码。

13. 故障快速索引

双击程序无反应或启动后退出

操作员：记录时间和弹窗，不要删除配置或关闭杀毒软件；尝试一次正常重启。

维护员：检查程序目录 logs/error、.NET Framework 4.7.2 运行环境、文件完整性和系统事件；安全软件白名单由IT审核配置。

初始化失败，出现“重试/跳过”

确认硬件上电和网络后点“重试”。普通生产禁止点“跳过”；“跳过”不会模拟硬件。连续失败时停机并保存提示截图。

PLC 或某个扫码器显示“未连接”

停止上料并记录具体设备；维护员在安全停机后检查供电、IP/端口、网线、交换机和配置。恢复“已连接”后必须做首件验证。

扫码重试耗尽、空码或条码无法解析

隔离工件；检查条码是否脏污、反光、破损、偏离ROI；检查码长、编码、分隔符和触发命令。不得复制上一件条码。

底板码与导向板码比对失败

按错配风险隔离，核对实物组合和数据库绑定；只有质量/工艺确认后，管理员才可修订绑定关系。

MES 超时、网络异常或结果 NG

停止放行、隔离在制品并记录主板码。检查网络和MES服务；不得关闭 SFC 来换取OK。恢复后防止同一工件重复查询/重复过站。

界面机械臂状态不更新

当前代码的机械臂流程未完整更新右侧状态字段，也不做MES报站。以机器人控制器、实体状态和通信日志为主，由软件工程师确认，不得仅凭该卡片放行。

治具次数达到 40 或需要清零

停止使用对应治具，按保养/更换流程处理。清零由PLC流程触发且会修改数据库；记录清零前值、底板码、原因、执行人和批准人。

刷新状态后 D110 不合理

同时查看PLC监控值、定时刷新值和通信日志。当前刷新逻辑的D110取值路径需要软件验证，不能因一次刷新为0就擅自改地址。

  • 能找到程序目录的 Config 和 logs，并能整理完整报修资料包。
