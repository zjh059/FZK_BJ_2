:: 1. 创建目录：适配Debug/Release路径
md "$(SolutionDir)FZK.Shell\bin\$(Configuration)\" 2>nul

:: 2. 复制DLL：同样适配Debug/Release路径
xcopy /Y "$(TargetDir)$(ProjectName).dll" "$(SolutionDir)FZK.Shell\bin\$(Configuration)" /S
xcopy /Y "$(TargetDir)$(ProjectName)\dll\*.dll" "$(SolutionDir)FZK.Shell\bin\$(Configuration)" /S 2>nul
xcopy /Y "$(ProjectDir)bin\$(Configuration)\dll\*.dll" "$(SolutionDir)FZK.Shell\bin\$(Configuration)"\ /S 2>nul

:: 调试用：输出复制结果（可选）
if %errorlevel% equ 0 (
    echo [成功] 已将$(TargetDir)dll\*.dll 复制到 $(SolutionDir)FZK.Shell\bin\$(Configuration)\dll
) else (
    echo [失败] DLL文件复制出错，错误码：%errorlevel% >&2
)

参数	作用
/Y	覆盖文件时不提示（必选，生成事件中无需交互）
/I	若目标不存在且复制多个文件，自动创建目标目录（推荐）
/S	复制非空的子目录（如果你的 dll 文件夹下还有子文件夹则需要，否则可省略）
2>nul	屏蔽 md 命令的 “目录已存在” 错误提示（避免生成事件输出冗余信息）
xcopy	Windows 系统的文件复制命令（支持文件夹递归、覆盖等）
/Y	静默覆盖目标文件（无需手动确认）
/S	复制非空文件夹和子文件夹（这里复制单个 DLL 时/S可省略，不影响功能）
$(ProjectDir)	Visual Studio 宏：当前项目的根目录（如D:\Project\ModuleA\）
$(SolutionDir)	Visual Studio 宏：解决方案的根目录（如D:\Project\）
$(ProjectName)	Visual Studio 宏：当前项目的名称（自动匹配项目名，避免硬编码）
$(TargetDir)	Visual Studio 宏：编译输出目录（自动匹配 Debug/Release，推荐优先使用）
